Cloudflare Pages — Workers & Pages → проект → Settings → Builds

1) Build command:
   npm run cf:build

2) Path / Build output directory (БЕЗ ведущего слэша):
   .vercel/output/static
   НЕ "/.vercel/output/static" — уберите "/" в начале, если поле это не принимает.

3) Deploy command (если поле ОБЯЗАТЕЛЬНО — так и нужно для Direct Upload / нового UI):
   npm run cf:deploy
   или напрямую:
   npx wrangler pages deploy .vercel/output/static

   НИКОГДА не используйте "npx wrangler deploy" (это Workers, не Pages).

   Если wrangler спросит имя проекта, добавьте (подставьте своё имя проекта в Cloudflare):
   npx wrangler pages deploy .vercel/output/static --project-name=claimgap

4) Non-production branch deploy command (если есть):
   можно такую же как в п.3, или оставить пустым если разрешено.

5) Version command:
   оставьте пустым или уберите, если не нужен отдельный шаг версионирования Workers.

6) NODE_VERSION = 20 (переменная среды сборки).

7) Settings → Functions → nodejs_compat.

После сохранения — Retry deployment.
